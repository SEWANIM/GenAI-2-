## 2. 🛠 제작 파이프라인

**기획 → 생성 → 검수 → 편집 → 통합** 순서로 진행

1. **기획**: 브랜드 아이덴티티 확정 → 스토리보드 작성 → 프롬프트 설계
2. **생성**: Midjourney(이미지) → Runway(영상화) → Suno(BGM)
3. **검수**: 톤·컬러·씬 일관성 확인, 불일치 시 재생성(vN)
4. **편집**: 씬 연결, 자막·내레이션 삽입, 컷 타이밍 조정
5. **통합**: 최종 렌더링 → 04_final 폴더 격리

## 3. 📁 파일 네이밍 & 폴더 정리 규칙

### 네이밍 공식
```
[프로젝트]_[씬]_[타입]_[버전].[확장자]
```
- 프로젝트: `NAERIRO` / 씬: `S1`·`S2`·`S3`
- 타입: `img`(이미지)·`vid`(영상)·`aud`(오디오) / 버전: `v1`, `v2`… (재생성 시 증가)

### 실제 예시
| 파일명 | 의미 |
|--------|------|
| `NAERIRO_S1_img_v2.png` | 새벽역 씬 이미지 2차 생성본 |
| `NAERIRO_S2_vid_v1.mp4` | 달리는 기차 씬 영상 1차본 |
| `NAERIRO_S3_aud_v1.mp3` | 일출 씬 BGM 1차본 |
| `NAERIRO_final_v3.mp4` | 최종 통합 영상 3차본 |

### 폴더 정리 구조
```
NAERIRO/
├── 01_planning/      # 스토리보드, 프롬프트 문서
├── 02_assets/
│   ├── images/       # Midjourney 생성 이미지
│   ├── videos/       # Runway 영상
│   └── audio/        # Suno 오디오
├── 03_edit/          # 편집 중간 파일
└── 04_final/         # 최종 렌더링본
```

### 재사용성 확보 원칙
- **버전 관리**: 재생성 시 원본 삭제 없이 vN으로 누적 → 이전 버전 복구 가능
- **타입 분리**: img/vid/aud 구분으로 도구별 결과물 빠른 검색
- **최종본 격리**: 04_final 폴더에 완성본만 보관해 혼동 방지

## 4. 💳 크레딧 제약 대응 전략
- **저해상도 프리뷰 우선**: 확정 전 저비용으로 구도 검증 → 확정 후 고해상도 1회 생성
- **시드(--sref) 고정**: 톤 일관성 유지로 재생성 횟수 최소화
- **핵심 씬 집중**: 크레딧을 S3(클라이맥스)에 우선 배분

## 5. 💬 심층 인터뷰 (요약)
- **도구 선택 근거**: 이미지 품질(MJ)·영상 자연스러움(Runway)·음악 맞춤성(Suno)의 조합이 감성 톤에 최적
- **프롬프트 수정 이유**: 초기 결과가 과하게 밝아 `blue hour`·`golden hour` 명시로 시간대별 색조 제어
- **불일치 보정**: 씬 간 색온도 차이 발생 시 편집 단계에서 컬러 그레이딩으로 톤 통일
